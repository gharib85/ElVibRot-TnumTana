Coding Guidelines

  • Indent with two spaces per indentation level. Do not use tabs
  • Inside subroutines, all strings must be declared with "len=*"
  • Separate subroutines/functions with two blank lines
  • At the very beginning of a subroutine, declare all dummy variables in one block. Align the intent declarations and the variable names in this block. Separate the block from the following declaration of local variables by a blank line
  • Always check the error code from allocations. Every call to allocate should be followed by a call to allocerror
  • Every routine and data structure must be documented
  • Every module must be appropriately tested
