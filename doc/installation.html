Compilation and Generation of the documentation

You can compile and generate the documentation of the Tnum-Tana program by using make while in the top folder which contains the makefile.

For compilation you have the following possibilities:

make clean install Clean up Compile and Install the documentation. The documentation will be installed in the ./doc folder. The excutable files (*.o and .mod) are generated in the ./point_o folder.
make clean Clean up files files generated from compilation: all the files of ./point_o and the files *.html of ./doc/reference .

For development, you can also use the following make targets:

make or make all Compile everything (but don't generate the documentation)
make clean Clean up files from compilation
make doc Generate the documentation in ./doc/
make work Compile the main program whose name (whitout the .f90) is affected to the flag WORKMAIN of the makefile. This will generate the executable file work.exe
make Tnum_MCTDH Compile the main program whose name is affected to the flag TNUMMCTDHMAIN of the makefile. This will generate the executable file Tnum90_MCTDH.exe
make Tnum to be written
make GWP to be written

The F90 compiler is used to compile *.f90 files. The F77 compiler is used to compile *.f files.

For information on how to compile a program using the library, see Using the Library

There are also additional details on the make process in the comments of the makefile.
