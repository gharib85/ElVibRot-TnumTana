Notes on the Input File

The purpose of the input file is to provide a description of the entire physical model and its numerical representation.

The input file consists of two main sections.

  • The first section provide the definition of the transformations which describe the numerical representation of the system. The project of the Tnum program started several years ago and has an historical evolution. There can exist two or severay ways to perform the same transformation. Therefore, in the firts section specify which routines will perform the defined transformations. The namelist syntaxis is used to list the variable of the section. It starts with "&" followed by the keyword variable (namelist name). It ends with "/". For example:
    &variable
    Starting line of the first section
    Old_Qtransfo
    Specify which routines will perform the transformations, Old_Qtransfo=t or Old_Qtransfo=f. By default Old_Qtransfo=t
    nb_transfo
    The number of transformations
    rho
    Enables to select the volume element
    Tana
    Enable to select if the keo will computed numerically or analytically. If Tana=t, the analytical keo is computed else it the numerical keo is computed. By default Tana=f
    /
    The end of the section
    For more details see an example here
    Each transformation is considered as a subsection of the first section. their number cooresponds to the value affected to nb_transfo. As in the firts section, the namelist synstaxis is considered in order to read the parameters of each aech transformation. For the definition of a given transformation, the first line of each transformation starts with &Coord_transfo followed by the keyword name_transfo ="name", the number of vectors nb_vect = n, the number of atoms nat , etc,.../ The possible names are:
    • identity : for the identity transformation
    • linear : for linear transformation
    • active: for the specification of the active coordinates
    • hyperspherical: for the hyperspherical transformation
    • poly: for the polyspherical transformation
    • bunch: for the bunch of vectors transformation
    • zmat: for the zmatrix transformation
    • flexible: "to be describe"
    • QTOX_ana: "to be describe"
    • order: "to be describe"
    • NM : "to be describe"
    • oneD : "to be describe"
    • linear_inv: "to be describe"
For more details, see here
